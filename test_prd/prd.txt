【文档版本】V1.0
【撰写日期】2025-11-21
【文档状态】正式发布

================================================================
1. 文档介绍
================================================================

1.1 背景说明
    B端产品通常面向企业内部用户或SaaS客户，相较于C端产品，B端登录功能更注重安全性、稳定性和效率，弱化炫酷的交互特效。
    本项目旨在规范B端产品的登录流程，提供账号+密码+验证码的通用登录方案，并包含必要的账号安全辅助功能（如修改密码）。

1.2 产品范围
    核心功能：用户登录（账号密码+图形/短信验证码）。
    辅助功能：修改密码、找回密码（可选）、账号安全策略。

1.3 用户角色
    - 普通用户：使用分配的账号登录系统进行业务操作。
    - 系统管理员：负责后台重置用户密码（针对忘记密码场景）。

================================================================
2. 功能需求详情
================================================================

2.1 登录功能
----------------------------------------------------------------
2.1.1 功能描述
    用户通过输入账号、密码及验证码进入系统。

2.1.2 页面元素
    1. 账号输入框：支持输入用户名/手机号/邮箱。
    2. 密码输入框：支持密文显示，提供“显示/隐藏”密码切换按钮。
    3. 验证码输入框：
       - 方案A（图形验证码）：右侧显示随机字符图片，点击可刷新。
       - 方案B（短信验证码）：右侧显示“获取验证码”按钮。
    4. 登录按钮：提交表单信息。
    5. 辅助链接：忘记密码？（视策略而定）、注册账号（仅SaaS版需要）。

2.1.3 交互逻辑
    1. 默认状态：
       - 登录按钮默认可用或置灰（取决于是否填写完整）。
       - 此时无任何错误提示。
    2. 输入交互：
       - 输入框获取焦点时：边框变色（如绿色或高亮蓝色），提示文字消失。
       - 输入框失去焦点时：恢复原状。
    3. 提交校验逻辑：
       - 点击【登录】按钮后，依次校验：
         a. 账号是否为空？ -> 提示：请输入账号。
         b. 密码是否为空？ -> 提示：请输入密码。
         c. 验证码是否正确/过期？ -> 提示：验证码错误或已过期。
         d. 账号密码匹配性？ -> 提示：账号或密码错误。
       - 校验不通过：在对应输入框下方显示红色错误文案。
       - 校验通过：跳转至系统首页。

2.2 修改密码功能
----------------------------------------------------------------
2.2.1 功能描述
    用户登录后，为了账号安全，可主动修改登录密码。

2.2.2 页面元素
    1. 原密码输入框：
       - 包含“明文/密文”切换按钮（眼睛图标）。
       - 包含“清空”按钮（仅在有输入内容时显示）。
    2. 新密码输入框：
       - 规则提示：如“6-18位，包含字母和数字”。
       - 包含“明文/密文”切换按钮。
       - 包含“清空”按钮。
    3. 确认新密码输入框：
       - 需与新密码一致。
    4. 验证码（可选安全加固）：
       - 手机验证码输入框 + 【获取验证码】按钮。
    5. 操作按钮：
       - 【取消】：返回个人中心/我的页面。
       - 【提交】：保存修改。

2.2.3 交互逻辑
    1. 验证码机制：
       - 点击【获取验证码】，按钮进入60秒倒计时（格式：60s后重新获取）。
       - 倒计时结束后恢复为“获取验证码”可点击状态。
       - 验证码有效期设定为60秒（或系统配置时间）。
    2. 密码可见性：
       - 鼠标按下“眼睛”图标：显示密码明文。
       - 鼠标松开或再次点击：恢复密文掩码（●●●●●）。
    3. 提交校验：
       - 原密码错误 -> 提示：原密码不正确。
       - 新旧密码相同 -> 提示：新密码不能与原密码一致。
       - 两次新密码输入不一致 -> 提示：两次输入的密码不一致。
       - 验证码错误 -> 提示：验证码错误。
       - 校验全部通过 -> Toast提示“重置密码成功”，并可能强制登出要求重新登录。

2.3 找回密码/重置密码
----------------------------------------------------------------
    1. 内部系统场景：
       - 界面通常不提供自助找回入口。
       - 逻辑：用户联系管理员 -> 管理员在后台重置为默认密码（如123456） -> 用户首次登录强制修改密码。
    2. SaaS/开放注册场景：
       - 提供“忘记密码”入口 -> 验证手机号/邮箱 -> 设置新密码。

================================================================
3. 非功能性需求 (NFR)
================================================================

3.1 安全性
    1. 密码存储：数据库中密码必须加密存储（如MD5+Salt或BCrypt），严禁明文存储。
    2. 防暴破限制：
       - 单一账号连续输错密码5次，锁定账号15分钟或需人工解锁。
    3. 会话管理：
       - 登录成功后生成Session/Token，长时间无操作（如30分钟）自动登出。
    4. 首次登录策略：
       - 若系统检测到用户使用初始密码登录，必须强制跳转至修改密码页面。

3.2 性能需求
    - 登录接口响应时间应小于500ms。
    - 支持高并发下的验证码发送与校验。

3.3 兼容性
    - 浏览器：支持Chrome、Firefox、Edge主流版本。
    - 分辨率：适配1366x768及以上分辨率。

================================================================
4. 数据埋点需求
================================================================
    1. 登录页PV/UV。
    2. 登录成功率/失败率（区分失败原因：密码错误 vs 验证码错误）。
    3. 修改密码功能的点击使用率。
