# 变更记录 001 — RAG 层升级：混合检索 + Re-ranking

- **日期**：2026-03-13
- **阶段**：Phase 1
- **涉及文件**：`core/rag_engine.py`、`requirements.txt`

---

## 一、变更背景

原始 RAG 检索仅使用 ChromaDB 向量相似度检索，存在以下问题：

1. 纯语义检索对关键词精确匹配弱（如专有名词、产品名）
2. 自写 `TextSplitter` 类约 80 行，逻辑冗余且未经工业验证
3. 检索结果无精排，Top-K 顺序不一定最优
4. 异常处理使用裸 `except:`，错误信息丢失

### 2.4 `__init__` 新增初始化

```python
# LangChain 文本分割器（替换自写 TextSplitter）
self.text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,
    chunk_overlap=100,
    separators=["\n\n", "\n", "。", "！", "？", ".", "!", "?", " ", ""]
)

# BM25 索引（关键词检索）
self._bm25_index = None

self._bm25_doc_ids = []

# BGE-Reranker 精排模型（加载失败时优雅降级）

    self.reranker = CrossEncoder('BAAI/bge-reranker-base')
except Exception as e:
    logger.warning(f"BGE-Reranker 加载失败，将跳过重排序: {e}")
    self.reranker = None
```

---

### 2.5 新增私有方法

#### `_build_bm25_index()`
从 ChromaDB 读取所有已存储的文本块，构建 BM25 倒排索引。每次 `add_knowledge()` 后自动重建。

#### `_bm25_search(query, top_k)`
对查询进行分词，在 BM25 索引中打分并返回 Top-K 文本块 ID 及得分。

#### `_rerank(query, candidates)`
用 BGE-Reranker（Cross-Encoder）对候选列表重新打分排序，返回精排后结果。

---

### 2.6 新增公共方法 `hybrid_search(query, top_k)`

三阶段检索流水线：

```
用户问题
   ↓
向量检索 Top-10 (ChromaDB cosine similarity)
   +
BM25 关键词检索 Top-10
   ↓
RRF 融合（向量权重 0.7，BM25 权重 0.3）
   ↓
BGE-Reranker 精排
   ↓
返回 Top-K 结果
```

**RRF 公式**：
```
score(d) = Σ weight_i / (rank_i(d) + 60)
```

## 六、测试过程中遇到的问题

### 问题 1：虚拟环境 Python 版本不兼容

**现象**：运行 `python -m venv .venv` 后激活，执行 `pip install` 报错：
```
Python 3.9 is not supported
```

**原因**：系统原有 `.venv` 是用 Python 3.9 创建的，Python 3.9 已被卸载，venv 无法使用。

**解决过程**：
1. 确认系统当前安装的是 Python 3.14
2. 用 `py -3.14 -m venv .venv_new` 创建新虚拟环境
3. 用户手动删除旧 `.venv`，将 `.venv_new` 重命名为 `.venv`
4. 重新激活新虚拟环境后依赖安装正常

---

### 问题 2：pyarrow 从源码编译失败（cmake 缺失）

**现象**：`pip install streamlit` 时 pyarrow 编译报错：
```
CMake must be installed to build from source
```

**原因**：Python 3.14 尚无 pyarrow 预编译 wheel，pip 尝试从源码编译，但系统未安装 cmake。

**解决过程**：
1. 先单独执行 `pip install pyarrow --prefer-binary`，强制使用二进制包（安装到 23.0.1）
2. 再安装其余依赖时 pyarrow 已满足，跳过编译

---

### 问题 3：chromadb 1.3.5 + Python 3.14 pydantic.v1 类型推断错误

**现象**：`import chromadb` 时抛出：
```
pydantic.v1.errors.ConfigError: field "chroma_server_nofile" not yet prepared so type is still a ForwardRef
```

**原因**：chromadb 1.3.5 内部使用 `pydantic.v1` 兼容层，该层在 Python 3.14 上无法正确推断某个字段的类型。

**解决过程**：
1. 确认 chromadb 1.5.5 已修复该问题
2. 将 `requirements.txt` 中 `chromadb` 版本从 `1.3.5` 升级到 `1.5.5`
3. 重新安装后导入正常

---

### 问题 4：BM25 零分测试用例存在逻辑漏洞

**现象**：测试 `test_bm25_search_zero_score` 中使用 `"无关词语"` 作为查询，预期所有文档得分为 0，但实际不为 0。

**原因**：BM25 按字符分词后，`"无"`、`"关"`、`"词"` 等单字在语料库（"登录"、"支付"等）中有字符级重叠，导致分数不为零。

**解决过程**：
1. 分析 BM25 分词机制：中文按单字切分，即 `list("无关词语")` = `["无","关","词","语"]`
2. 语料中虽无 "无关词语" 整词，但个别字符可能出现在其他文本中
3. 将查询改为纯 ASCII 字符串 `"xxxxxqqqqqzzzzz"`，确保与中文语料库完全无交集
4. 修改后测试通过，分数全部为 0.0
